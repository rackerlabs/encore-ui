Date/Time Formatting

Date Formats

Short-form

  • YYYY-MM-DD
  • This matches the date format international standard, ISO 8601.
  • Example: 2016-03-25

Long-form

  • MMMM D, YYYY
  • Example: March 25, 2016

Date Formats (Month Only)

Micro Month Format

  • MM / YY
  • This is the accepted standard format for credit card expiration date, which is honored in Encore, matching the format of physical credit cards as closely as possible.
  • Note those spaces between the slash!
  • Example: 03 / 12

Short-form

  • MM / YYYY
  • Example: 03 / 2016

Long-form

  • MMMM YYYY
  • Example: March 2016

Time Formats

Short-form

  • HH:mmZ
  • The Z above is a colon-separated timezone offset (-10:00, not -1000) for explicit hour/minute offsets.
  • Example: 22:56-06:00

Long-form

  • h:mmA (UTCZ)
  • Example: 10:56PM (UTC-06:00)

DateTime Formats

Long-form

  • MMM D, YYYY @ h:mmA (UTCZ)
  • This matches the Long-form Date Formats and Long-form Time Formats with the " @ " between.
  • This format is seen the most in Encore Cloud to show create/update timestamps.
  • Mar 25, 2016 @ 10:56PM (UTC-06:00)

Short-form

  • YYYY-MM-DD @ HH:mmZ
  • This matches the date format international standard, ISO 8601.
  • 2016-03-25 @ 22:56-06:00

Relative Dates

  • Relative dates, such as such as "4 hours ago" or "1 week ago", are reserved for more informal settings like user notes or comments.
  • Relative dates should not extend too far into the past. For instance, "1 week ago" may be helpful to the user, but "8 months ago" or "3 years ago" is better represented by the non-relative format detailed above, as it reveals more specific information about dates that provide better context and information to the user when comparing dates and times.

rxOptionTable
directive in module rxOptionTable

Deprecated API This directive will be removed in EncoreUI 4.0.0

Description

This directive is used to build a table of radio/checkbox inputs.

Usage

as element:
<rx-option-table
       field-id="{String}"
       model="{Object}"
       type="{String}"
       empty-message="{String}"
       data="{Array}"
       columns="{Object}"
       [selected]
       [required]
       [disable-fn="{Function}"]>
</rx-option-table>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
fieldIdString

Used as the base for unique identifiers within the generated markup.

modelObject

The AngularJS model to tie all radios/checkboxes together.

typeString

(radio | checkbox) Type of input to be used.

emptyMessageString

A default message if the data attribute is empty.

dataArray

Array of objects used to populate table. Properties must match column keys. For checkboxes, checked values default to true unless value and falseValue attributes are given.

Example: 

[
    {
        'name': 'Option 1',
        'value': 0
    }, {
        'name': 'Option 2',
        'value': 1
    }, {
        'name': 'Option 3',
        'value': 2
    }
]
columnsObject

Array of column data to match against data objects. Each object may include the following properties.

PropertyTypeDetails
labelString

Column display value

keyString

Object key used to display data from the data object

selectedLabel
(optional)
String

(optional) Label to display alongside preseleted-values. Expressions are allowed; see demonstration samples.

Example:

[{
    'label': 'Name',
    'key': 'name'
}]
selected
(optional)
String

Array of objects to match against data for preselection on page load. If excluded, no values will be preselected on initial load.

required
(optional)
Boolean

Value passed to input's ng-required attribute. For checkboxes, a true value means that there must be at least one checkbox selected.

disableFn
(optional)
Function

Optional callback function to determine if option should be disabled. Parameters tableId, fieldId, and rowId will be passed to the function.

Example:

<rx-option-table disable-fn="disableOption(tableId, fieldId, rowId)"></rx-option-table>
